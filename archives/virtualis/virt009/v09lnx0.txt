  Este documento foi escrito para pessoas que acabaram de instalar o
  Linux e não sabem o que fazer a seguir. A maioria dos comandos
  discutidos aqui deve funcionar em todas as distribuições do Linux mas
  já que eu uso a Red Hat 5.0, alguns podem ser específicos a ela. Já
  usei o Caldera OpenLinux 1.3 e inclui alguma informação específica
  para esta distribuição. Se algum de você tiver alguma sugestão ou
  idéia para melhorar este documento, seja bem-vindo. Todos os comandos
  estão entre aspas e você deve digitá-los sem aspas. Por exemplo, se
  você ver digite "ls" então você deve digitar ls. Você também deve
  pressionar ENTER após digitar cada comando. Existem alguns comandos
  úteis no documento mas para a referência completa sobre o comando
  você deve consultar documentos adicionais.

  Vamos começar "bootando" o Linux. Quando você "boota" o Linux, verá
  uma porrada de mensagens aparecendo. Você não precisa entender todas
  agora mas se você tiver alguns erros durante o "boot", você deve
  dar uma olhada. Isso ajuda ajuda a entender o motivo do erro. A
  primeira coisa que você deve fazer é se logar. No prompt digite
  "root" (ou qualquer usuário que você já tenha criado) e depois
  digite a senha. A senha é especificada durante a instalação. Se
  você instalou o Linux na sua máquina então você é o usuário root
  e tem acesso de administrador do sistema. Se você não escolheu 
  uma senha então o sistema não lhe pedirá nenhuma. Você então chegou 
  no prompt do Linux. O prompt será um # se você for root ou um $ se 
  for outro usuário e tenha escolhido o shell BASH. Se você é iniciante
  com o Linux então você deve usar o shell BASH. Dentre os vários
  shells do Linux, eu prefiro usar o BASH pois é fácil de usar. BASH
  também é o shell default da maioria das distribuições. Se prompt
  deve se parecer com algo assim.
 
  [root@yoom.com /root]#

  Se precisar sair (logout) digite "exit".

  Uma vez que você tenha se logado digite "dmesg" para ver as mensagens
  do boot. Você deve ver algo parecido com o seguinte:

  Você vai notar que as mensagens passam antes que você possa lê-las.
  Para vê-las página por página digite "dmesg | less" ou "dmesg | more".

  O comando dmesg fornece informações valiosas sobre periféricos 
  detectados pelo Linux. Ele também ajuda a descobrir se houve algum
  problema. Como você vê na linha sb: Interrupt teste on IRQ5 failed -
  device disabled. Isso significa que houve um problema na configuração
  da placa de som na IRQ5. Se você receber tais erros, pode ser que
  algo em seu hardware não esteja funcionando corretamente sob o Linux.

  O shell BASH é bem fácil usar. Se você gosta de trabalhar bastante na
  linha de comando, o achará muito fácil. O shell bash permite usar o
  comando anterior pressionando-se a tecla de seta para cima. Você
  também pode procurar por comandos anteriores ao teclar "CTRL-R" e
  digitar algumas palavras dos comandos anteriores. Para limpar a tela
  pressione CTRL-L ou simplesmente digite "clear".

  Outro comando importante é df. Digite apenas "df" e verá algo assim:

  Filesystem      1024-blocks     Used    Available Capacity      Mounted on
  /dev/hda6        388362         341804  26501   93%             /
  /dev/hda5        614672         572176  42496   93%             /dosd

  Ele lhe informa sobre todas as partições montadas, espaço disponível
  e espaço usado. O espaço mostrado é de 1024 blocos que é 1024 bytes 
  ou um KiloByte. Ele também mostará em qual diretório a partição está 
  montada. Como no DOS e no Windows as partições e dispositivos são 
  colocados em letras como C:, D:, E:; no Linux, as partições ou 
  dispositivos são montados em diretórios. Por exemplo /dev/hda5 está 
  montado em /dosd. Normalmente /dosc, /dosd, podem ser partições dos 
  montadas. Mas poderia ser qualquer outra coisa. O que significa que 
  você pode acessar seus arquivos DOS através do Linux indo até estes 
  diretórios.
  
  Outro comando útil é ls. Digite "ls" e você verá algo assim:

  bin/ dev/ etc/ lost+found/ proc/ tmp/
  boot/ dosc/ home/ mnt/ root/ usr/
  cdrom/ dosd/ lib/ opt/ sbin/ var/

  Digite "ls -l" para ver um lista mais completa. Ele lhe mostrará os
  donos (owners), permissões, data e hora da última modificação e 
  tamanhos dos arquivos. Você precisará entender o permissionamento de 
  arquivos para sacar o funcionamento das operações básicas no Linux. 
  Permissões são úteis no sistema multiusuário Linux onde você precisa 
  restringir ou permitir o acesso a arquivos ou diretórios.
  
  drwxr-xr-x 2    root root 2048  Sep 17  12:49   bin/
  drwxr-xr-x 2    root root 1024  Oct 4   23:24   boot/
  drwxr-xr-x 2    root root 1024  Sep 2   17:32   cdrom/
  drwxr-xr-x 3    root root 21504 Oct 22  12:54   dev/
  drwxr-xr-x 2    root root 1024  Oct 2   21:59   dosc/
  drwxr-xr-x 13   root root 21504 Jan 1   1970    dosd/
  
  O comando cd é usado para mudar o diretório. Você pode tentar 
  digitando "cd /" para ir até o diretório raiz. Digite "cd -" para 
  voltar para onde você estava. Se você digitar apenas "cd", irá 
  retornar ao seu diretório home. 


  Instalando softwares e abrindo arquivos compactados no Linux
  ------------------------------------------------------------

  Se você fizer o download de documentos, utilitários, softwares ou 
  qualquer outra coisa para Linux, verá que vários deles tem as 
  extensões .tgz ou .tar.gz. Neste caso você deve digitar o seguinte 
  comando para extrair os arquivos. Substitua nome_do_arquivo.tar.gz 
  com o nome do arquivo que você deseja.

  gzip -dc nome_do_arquivo.tar.gz | tar xvf -

  Se você fizer o download de arquivos Linux no DOS, pode ser que os
  nomes de arquivos fiquem truncados. Neste caso, você terá que 
  renomear seus arquivos antes de extraí-los no Linux. Para renomear 
  arquivos digite "mv arq_antigo arq_novo". Substitua arq_antigo com o 
  nome atual do arquivo e arq_novo com o nome do arquivo que você quer 
  que o arquivo tenha.

  Vários arquivos estão também no formato .rpm. Este formato é usado 
  pelas distribuições Red Hat e Caldera e também podem ser usado em 
  outras distribuições. Para instalar o pacote rpm digite:

  rpm -i filename.rpm

  Se estiver fazendo um atualização (upgrade) de um software, digite:
 
  rpm -U filename.rpm

  Se sua distribuição não suporta RPM, você pode instalá-lo. Procure o
  RedHat Packet Manager (RPM) por aí. De maneira igual, existe o 
  pkginstall em algumas distribuições como Slackware, para gerenciar
  arquivos .tar.gz.


  Cara ! O que é man ? São páginas de help ou manuais que nos ajudam
  a obter mais informações sobre um comando. Para conseguir o ajuda
  sobre o man, digite "man man". Da mesma maneira, para conseguir ajuda
  num determinado comando, como rpm, digite "man rpm". Para conseguir
  ajuda no comando ls, digite "man ls" e por aí vai. Você pode 
  conseguir ajuda em todos os comandos usando man. Para começar, 
  procure ajuda nos comandos mais usados. Estes comandos irão lhe 
  ajudar a trabalhar com arquivos e diretórios. Alguns comandos são:


  cat      para mostrar o conteúdo de um arquivo 
  cp       copia arquivos
  du       para verificar o espaço em disco utilizado
  pine     cliente de email
  find     procura arquivos no sistema linux
  grep     procura palavras em um arquivo ou comando
  kill     para matar qualquer processo; ps para ver o número do 
           processo
  less     se você der um "cat" num arquivo pode redireciná-lo (com | )
           para ver página a página
  ln       cria ou remove links entre arquivos e diretórios
  lpr      imprime arquivos ou saídas em uma impressora 
  ls       lista arquivos ou diretórios
  mkdir    cria um novo diretório
  more     igual a less mas less ainda é melhor !
  mount    veja os periféricos montados
  umount   desmonte volumes montados
  mv       move ou renomeia um arquivo
  passwd   altera sua senha
  ps       veja os processos que estão rodando
  rm       remova arquivos ou diretórios
  rmdir    remova diretórios
  useradd  inclua um usuário no sistema
  userdel  exclua um usuário do sistema
  usermod  modifique um usuário do sistema
  which    descubra onde um programa está instalado
  who      veja os usuários que estão logados
  zless    veja o conteúdo de um arquivo .gz (comprimido)


  Mais  algumas  dicas  para  usuários do bash. Se você sabe a primeira
  letra de um comando e não sabe o resto, digite "a" e  então pressione
  TAB duas vezes e o bash irá mostrar uma lista de possibilidades. Você
  também pode pressionar TAB apenas uma vez para completar o comando se
  há  apenas  uma  possibilidade. Isso  economiza  um  bom  tempo  de 
  digitação. Por exemplo, digite  "mou" e  então  digite TAB; o  bash 
  completará e monstrará "mount" na linha de comando.
 
  Pressionando TAB duas vezes, será mostrada uma lista de todos os 
  comandos Linux. É algo assim:

  There are 1212 possibilities. Do you really wish to see them all? (y or n)
  (Existem 1212 possibilidades. Deseja mesmo ver todas? (y ou n))

  Digite "y" (yes=sim) e você verá todos!

  Às vezes, após digitar um comando, a tela corre muito rápido para que
  você possa ler (a não ser que você tenha superpoderes). Neste caso, 
  você pode ver a tela anterior pressionando Shift e PageUp juntas.

  Se você digitar alguns comandos, você pode parar pressionando CTRL-C 
  ou ESC. Isso pode não funcionar no man ou less, nestes casos, digite 
  apenas "q". Se você precisa editar alguns arquivos tente o pico ou 
  joe. Estes dois são muito fáceis de usar. Joe funciona mais ou menos 
  como o bom e velho WordStar e pico é o editor para o Pine. Usuários 
  experientes podem tentar vi ou emacs. Estes dois são editores muito 
  poderosos mas tem uma curva de aprendizado muito ingreme. Exemplos: 
  "joe nome_do_arquivo". Substitua o nome_do_arquivo pelo nome que você
  realmente deseja.
  
  A maioria das distribuições instalam o X-Window. Para iniciar o 
  X-Window digite "startx". X-Window é uma GUI (interface gráfica). 
  Existem vários sabores disponíveis, cada um com características 
  diferentes. Se você está usando o Caldera, digite "lisa". Você 
  também pode configurá-lo através de uma interface gráfica no 
  X-Window.

  A maioria dos usuários podem querer usar alguns disquetes ou 
  partições DOS. Você pode digitar alguns comandos DOS no Linux sem 
  mesmo montar seus discos. Digite "man mtools" para ver uma lista 
  destes comandos. Estes comandos começam com "m". Por exemplo, o 
  comando DOS copy, ficaria como mcopy. De maneira similar, existem 
  vários comandos como mattrib, mcd, mcopy, mdel, mdeltree, mdir, 
  mformat, mlabel, mmd, mrd, mmove, mren, mtype, mzip, etc. Para ver 
  mais documentação Linux procure debaixo dos seguintes diretórios. 
  Se os arquivos possuem a extensão .gz, para vê-los digite 
  "zless nome_do_arquivo.gz", trocando nome_do_arquivo pelo nome
  do arquivo que você deseja mesmo ver.
