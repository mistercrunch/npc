  A maioria dos diretórios que guardam arquivos Linux são padronizados.
  Outros sistemas UNIX possuem diretórios idênticos ou parecidos. Veja
  quais são os diretórios mais importantes:

  /
  Este é o diretório raiz (root). Contém o todo o sistema Linux e todos
  seus subdiretórios. Não encha este diretório com seus arquivo !!!

  /home
  Este diretório contém os diretórios 'home' dos usuários. Em outros
  sistemas UNIX, ele pode ser o diretório /usr ou /u.

  /bin
  Este diretório contém vários programas e utilitários do Linux. bin é
  a abreviação de 'binários', arquivos executáveis inteligíveis apenas
  para o computador.

  /usr
  Este diretório guarda diretórios voltados aos usuários. Alguns dos
  mais importantes estão descritos nas seções seguintes. Outros
  diretórios encontrados em /usr incluem:

    +-------+-------------------------------------+
    | docs  | vários documentos, incluíndo várias |
    |       | informações úteis sobre o Linux     |
    +-------+-------------------------------------+
    | man   | páginas 'man'                       |
    +-------+-------------------------------------+
    | games | jogos !                             |
    +-------+-------------------------------------+

  /usr/bin
  Este diretório guarda programas voltados ao usuários, i.e., programas
  de terceiros instalados pelo usuário (e.g., Netscape, jogos, etc).

  /var/spool
  Este diretório possui vários subdiretórios. mail guarda seus arquivos
  aqui, spool coloca os arquivos a serem impressos, e uucp guarda os
  arquivos copiados entre as máquinas Linux.

  /dev
  Linux trata tudo como arquivo ! O diretório /dev armazena dispositivos.
  São arquivos especiais que servem como 'portas' para dispositivos
  físicos (periféricos) do sistema. Por exemplo, se você copiar alguma
  coisa para /dev/fd0, estará na verdade enviando dados para o drive de
  disquetes do sistema. Seu terminal é um dos arquivos /dev/tty.
  Partiçäes do seu disco rígido estão como /dev/hd0. Mesmo a memória
  RAM do sistema é um dispositivo ! Um dispositivo famoso é /dev/null,
  também conhecido como 'bit bucket' (balde de bits). Toda informação
  enviada para /dev/null é descartada. 

  /usr/sbin
  Este diretório guarda arquivos de administração do sistema. Se você
  der um ls -l, verá que você deve ser o 'owner', ou root, para
  executar estes um destes comandos.

  /sbin
  Este diretórios guarda arquivos do sistema que geralmente são
  executados automaticamente pelo Linux.

  /etc
  Este diretório e seus subdiretórios guardam vários arquivos de
  configuração do Linux. Estes arquivos geralmente são arquivos texto,
  e podem ser editados para mudar a configuração do sistema (tenha
  certeza do que você está fazendo!)
