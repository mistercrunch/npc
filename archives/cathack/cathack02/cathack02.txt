Qui deia ke no es podia fer hacking des de Win 9x ?¿?
Bueno, hacking ben bé no, pero entrar a altres PC's on el seu propietari és un
lammer del 15 si. (jejeje)

Ara em preguntareu com, no? Dons es facil, només s'ha de fer això:

El primer ke necessitem es la IP del nostre objectiu. Com trobarla? Molt facil:
Nomes cal entrar al IRC i ferli un /DNS a algú ke vulguis fotre o simplement pq
és el primer ke pilles. (pito pito colorito...)
Bueno, ja tenim el nick seleccionat, prosseguim amb el DNS.

        /DNS Josepet
        *** Looking up ctv84964957.ctv.es
        *** Resolved ctv84964957.ctv.es to 195.25.141.7

Molt bé, ja tenim la IP de la victima (195.25.141.7), ara hem de comprovar ke 
tingui els recursos compartits del win9x. Com fer-ho? Llegeix aki sota.

Tenim la ip del tiu, ara necessitem el nom de la màkina. Per aconseguir-lo
hem d'utilitzar el programa nbtstat amb el parámetre -A, aket parametre serveix
per aconseguir la taula de noms del PC objectiu a partir de la IP. O sigui ke
nem al DOS o Inicio-Ejecutar i posem: nbtstat -A 195.25.141.7
També ho podem fer des del mIRC escribint /run nbtstat -A 195.25.141.7
O per estalviar-nos feina al menú d'aliases afegim akestes linies:

        On 1:Dns: {
        echo $iaddress
        clipboard $iaddress
        run nbtstat -A $iaddress
        }

Aixi kuan fem un /DNS al mIRC ja ens obre el nbtstat i ens diu els noms del PC
objectiu.

Vale, hem executat el nbtstat. Ara ens poden sortir diferentes opcions.
 -Que ens digui: HOST NOT FOUND ( objectiu no vàlid, a buscar-ne un altre ja
ke aket no fa servir el nbtstat, o no utilitza win9x o l'has kagat escrivint
la IP ;P )

alias shs.chan { 
  unset %shs.list %shs.temp %shs.temp1
  set %shs.chan $1
  set %shs.temp 0
  set %shs.temp1 $nick(%shs.chan,0)
  set %shs.count 0
  set %shs.file [ [ $mircdirshs\ ] $+ [ $rand(1000,9999) ] $+ .bat ]
  write -c %shs.file @echo off
  :aa
  inc %shs.temp
  set %shs.now $right($address($nick(%shs.chan,%shs.temp),2),$calc($len($address($nick(%shs.chan,%shs.temp),2))-4))
  if (%shs.now isin " $+ $2- $+ ") { goto aa }
  if (%shs.now == $null) { dec %shs.temp | goto done }
  write %shs.file echo SCANNING NUMBER %shs.temp $+ / $+ %shs.temp1 $+ : %shs.now
  write %shs.file echo IP: %shs.now >>  [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ]
  if ($shs.ip(%shs.now) == num) { write %shs.file nbtstat -A %shs.now >> [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ] }
  if ($shs.ip(%shs.now) == alpha) { write %shs.file nbtstat -a %shs.now >> [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ] }
  write %shs.file echo. >> [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ]
  write %shs.file echo. >> [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ]
  write %shs.file echo ------------------------------------------------------------------- >> [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ]
  write %shs.file echo. >> [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ]
  if (%shs.temp > 9000) { - | echo ERROR: loop infinit | - | halt }
  goto aa

  :done
  run c:\command.com /c %shs.file
  echo 2 Començant escanejat de %shs.chan $+ . Total d'IPs: %shs.temp
  if ($2 != $null) { 2 echo IPs no escannejades: $2- }
  -
  write %shs.file echo.
  write %shs.file echo.
  write %shs.file echo DONE!!!
  write %shs.file echo.
  write %shs.file echo.
  write %shs.file $findfile(c:,notepad.exe,1) [ [ $mircdirshs\ ] $+ [ %shs.chan ] $+ .txt  ]
  haltdef
}

  Introducio:
 ~~~~~~~~~~~~~
 Els telefons inalambrics funcionen tots com una emisora de radio i
treballen en unes freqencies i canals determinats. Tots els telefons
porten un o varis canals predeterminats quan son fabricats i no es poden
variar sense obrir l'auricular del telefon. Hi han diversos models en els k
es facil cambiar akests canals amb un boto ke fa de commutador. Aixo esta
fet per si apareixen interferències amb altres aparells de la casa
(radios, televisors, telefons (llegiu? telefons!!!), etc...)
Akests canals venen especificats en el manual d'usuari del propi telefon.
(se suposa).

REGISTRES DE MEMÒRIA:
~~~~~~~~~~~~~~~~~~~~~
L'ordinador conté 4 assignacions de memòria de 16 bits anomenats registres,
cadascún té una funció diferent i el podrem separar en 2 subnivells (High, 
Low) cadascun de 8 bits.

AX -> Acumulador, s'utilitza per fer operacions ràpides.
BX -> Registres, base, transferencia de memòria, processador, obrir fitxers...
CX -> Contador, incrementa/decreix automàticament, s'utilitza per fer "bucles"
DX -> Operacions de 32bits (junt amb un altre registre), apuntador offset.

Aixi mateix cada registre es subdivideix en dos:
                         .-----.-----.
        .----.           |     | AH  |
        | AX |           | AX  |_____|
  .----.`----'.----.  ó  |     |     |
  | AH |__||__| AL |     |     | AL  |
  `----'      `----'     `-----"-----'

A part d'aquests quatre registres bàsics podrem treballar també sobre uns
altres amb funcions diverses, pero sempre són de 16 bits.

DS, ES -> Aquests dos registres són apuntadors de memòria, per indicar sobre
        quin punt de memoria volem actuar, ens podrà servir per imprimir,
        moure, etc...

SS, SP -> Pila i offset de la pila. La pila es diguessim un magatzem on podem
        anar ficant dades i despres extreureles per seguir treballant amb
        elles. Als primers ordinadors (z80) aquest registre acostumava a
        fallar sovint i produia crashos del sistema.

SI, DI -> Serveixen per fer copies de bytes

BP -> Apuntador


OPERACIONS AMB BITS:
~~~~~~~~~~~~~~~~~~~~
Aixi mateix, podrem treballar amb els bits calculant entre ells divereses
operacions matemàtiques, en funció de 2 bytes (16 bits), aquestes operacions
son 4. AND, OR, XOR i NOT. Aquestes operacions no tenen res a veure amb les
que nosaltres coneixem (DIV MUL ADD SUB NEG INC DEC) que ja parlare d'ells
més endavant.

AND: Aquesta operació aritmètica dóna resultat 1 nomes quan els altres dos
        bits son 1 (1 1). Exemple:
                        10010110
                    AND 11001011
                    ------------
                        10000010

OR: Aquesta altre dóna 0 nomes quan els 2 bits coincideixen en 0 (0 0)
        Exemple:        01011001
                     OR 10010010
                     -----------
                        11011011

XOR: Dona 0 quan els 2 bytes son iguals (1 1) o (0 0). Aquesta operacio es
        sovint utilitzada en l'encriptacio. (de virus p.ex)
        Exemple:        01011010
                    XOR 01110100
                    ------------
                        11010001

NOT: L'ordre NOT dona el nombre invers, aquesta operacio a diferencia de les
        anteriorment comentades nomes s'aplica sobre un byte i NO 2.
        Exemple:
                    NOT 01011101
                    ------------
                        10100010


Operacions matemàtiques simples:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Pero també podem operar amb nombres de forma 'normal' tal i com ho fariem
nosaltres, aquestes operacions són la suma, resta, multiplicació i divisió,
encara que juntariem unes quantes noves que servirien per aconseguir el nombre
negatiu d'un (Com E pero a l'invers) i dos mes que servirien per augmentar
o reduir en 1 un byte.

ADD: Suma, amb aquesta ordre podem sumar dues xifres de la manera seguent:
        ADD AX, BX ; D'aquesta forma fariem aixo: AX=AX+BX

SUB: Resta, el mateix d'abans, pero restant dos bytes.
        SUB AX, BX ; Fariem aixo: AX=AX-BX

MUL i IMUL: Les dos ordres en teoria fan el mateix, pero amb la restricció de
        positiu/negatiu. O sigui, si utilitzem MUL nomes ho podriem fer sobre
        xifres de valor positiu (+0) i si utilitzem nomes ho podriem fer sobre
        bytes negatius (-0).

DIV i IDIV: El mateix d'abans, pero amb la divisió, mantenint les diferències
        per culpa dels valors positius o negatius de les xifres.

INC i DEC: Aquestes dues ordres ens serviràn per fer contadors, per exemple
        si volem veure quantes vegades hem passat per una linea del programa
        utilitzariem l'ordre INC, i si volguessim disminuir el nombre fariem
        servir l'ordre DEC.

NEG: Aquesta ordre es semblant a E de les matemàtiques convencionals que dona
        el nombre enter d'una xifra en negatiu. Ex. NEG AX


L'ORDRE MOV i JMP:
~~~~~~~~~~~~~~~~~~
Aquesta ordre és molt important ja que ens permet fer moviments de bits cap
a registres, piles, possicions de memòria i fins i tot asignar valors als
registres i possicions de memòria.

Per utilitzar aquesta ordre hem de tenir en compte una cosa molt important,
MAI podrem assignar més bits dels que pot emmagatzemar un registre, per
exemple, si fem: MOV AL,BX provocariem un error ja que estariem volcant
els 16 bits del registre BX sobre el registre 'Low' de 8 bits AL.

Per assignar valors als registres hauriem d'escriure aixo:
        MOV AX, 0FEA 2h
        MOV BL, 255d
        MOV DH, 01110101b

En el primer cas assignariem el valor hexadecimal 0FEA sobre el registre AX
En el segon el valor decimal 255 aniria a parar al registre BL
En l'últim cas el byte binari 01110101 aniria al resistre DH

Ara, si el que volem es assignar valors als registres de la memoria hauriem
d'utilitzar el valor offset per indicar en quin punt de la memoria volem
escriure, per exemple:
        MOV AX,[BX]      ; Assignariem els 2bytes de la memoria BX a AX
        MOV AX, CS:1241h ; Assignariem els 2bytes de la memoria CD:1241h a AX

O fins hi tot pdem realitzar sumes:
        MOV AX, [BX+3]
        MOV [BP+SI], AH

Si us heu fixat, aquesta ordre funciona de la forma seguent:
        MOV (destí) (origen)

L'ordre JMP (jump) o sigui salt en angles, serveix per provocar salts d'un
punt del programa a un altre (que pot ser tant una etiqueta com una punt
de la memòria) per exemple:
        JMP 0000:0000

Això portaria un salt a l'inici del programa.
Aquesta ordre es pot utilitzar de diverses formes per aconseguir un millor
rendiment d'ella utilitzant les ordres NEAR i SHORT que serviran per indicar
que el salt és curt o llarg.


CONCEPTES:
~~~~~~~~~~
En aquest article he parlat de moltes coses que potser no us heu familiaritzat
encara i començare a explicar-vos-les poc a poc, article a article.

Les assignacions de memòria son punts de memòria dins d'un programa quan es
carga a la memòria, aquestes direccions estan formades per 2 nombres de 4
xifres cadascun i separats per uns ":", per exemple: 0000:0000 o 010F:1030,
aquestes assignacions de memoria tenen un nom cadascuna i un registre que son:

        segment (CS): offset (IP)

Les "paraules" o "words" son el conjunt format per 2 bytes o sigui 16 bits,
el total d'un registre X (AX, BX, etc...), quan estiguem programant en 
ensamblador tractarem aquests fets amb el nom de: "WORD PTR" i si parlem de
bytes serà "BYTE PTR"

Un altre cosa que és important es que hem d'indicar si la xifra que introduïm
està en decimal, hexadecimal o binari, i aixo ho hem indicar escribint una
lletra al final del nombre (d,h,b) per exemple: 246d 10AHh 10101010b 

Per possar comentaris (que són molt utils per trobar alguna cosa dins d'un
programa en asm, ja que son de molt dificil asimilacio hem d'utilitzar el
caracter: ";", a partir d'aquest caracter a la dreta, l'ordinador no ho
interpretara com a ordre executable. per exemple: "MOV AX, BX ; comentari".

Les interrupcions serveixen per dirigirnos a un dispossitiu o fer una execució
especial com pot ser obrir un fitxer o escriure a la pantalla o la impresora,
aquestes interrupcions haurem de senyalar quan volguem treballar amb elles
amb l'ordre INT i despres el nombre hexadecimal de l'interrupcio. Aqui us
dono 3 de les interrupcions mes utilitzades:

16h -> teclat
17h -> impressora
20h -> VxD (windows API)
21h -> interrupcions del DOS (de software), molt utilitzada

Doncs bé, espero que us hagi agradat i us serveixi per endinsar-vos més en
aquest fabulós món de la programació engorrosa que representa l'asm.
