01- Sn1ff1nG faQ
+-+-+-+-+-+-+-+-

esse faq so tem fins educacionais

o q eh um sniffer?
sniffer eh um programa q fica numa rede,ou computador,como se fosse um
grampo telefonico,um sniffer pode ser usado com 2 fins:
comercial: pra ajudar na manuntencao de uma rede por exemplo
criminoso: pra ouvir conversa dos outros,pegar senhas, etc...

aqui vo falar sobre o metodo "criminoso"

como eu posso evitar q saibam o q eu estou falando?

a melhor defesa eh criptografia,algumas tecnicas pra vc usar sao:
SSL (Secure Sockets Layer): pra encriptar seus dados q trafegam
por http como cartao de credito,etc...

PGP (Pretty Good Privacy): pra encriptar seus emails,e nao deixar
admins bundoes lerem .

ssh (Secure Shell): o ssh se tornou padrao de uns tempos pra ca
pra vc logar em algum sistema UNIX,por exemplo,por telnet seus dados
nao sao encriptados,deixando qquer um ver,ja por ssh vai tudo encriptado.

como eu posso sniffar os outros?
existem varios programas pra varias plataformas q fazem isso como:

Windows
-------
Network Associates Sniffer- http://www.nai.com
BlackICE Pro- http://www.networkice.com/
CiAll- http://members.tripod.com/~radhikau/ciall/ciall.htm
SocketSpy32- http://www.win-tech.com
windump- http://netgroup-serv.polito.it/windump/

UNIX
----
tcpdump- ftp://ftp.ee.lbl.gov/
sniffit- http://reptile.rug.ac.be/~coder/sniffit/sniffit.html
esniff- http://www.phrack.com/search.phtml?view&article=p45-5
exdump- http://exscan.netpedia.net/exdump.html

quais protocolos sao vulneraveis?

Telnet e rlogin (vai tudo descriptado incluindo password)
http (se o site usar autenticacao "basic" o 
      password nao vai criptografado!)
snmp (se o protocolo usado for SNMPv1 
      o password tb nao vai criptografado!)
nntp (o password nao vai criptografado!)
pop (o password nao vai criptografado!)
ftp (o password nao vai criptografado!)
imap (o password nao vai criptografado!)

ja deve ter dado pra entender + ou -
se quiser algo mais completo:
http://www.robertgraham.com/pubs/sniffing-faq.html

02- awe64 + fr33bsd
+-+-+-+-+-+-+-+-+-+

Eu escrevi essa materia, porque tive pequenas dificuldades em configurar 
minha placa de som
no FreeBSD (um sabor do unix), entao resolvi escreve-la pra facilitar a 
vida de outras pessoas 
que possam ter os mesmos problemas que tive. Vamos la:
-Coloque as seguintes linhas em seu kernel:
controller pnp0 
controller snd0 
device sb0 at isa? port 0x220 irq 5 drq 1 
device sbxvi0 at isa? drq 5 
device sbmidi0 at isa? port 0x330 
device opl0 at isa? port 0x388 
device awe0 at isa? port 0x620 
-Recompile seu kernel, caso nao saiba, de uma olhada em 
http://www2.iqm.unicamp.br/~g950474/
-Crie um arquivo chamado kernel.conf no diretorio /boot e coloque as 
seguintes linhas nele:
pnp 1 0 os enable irq0 5 drq0 1 drq1 5 port0 0x220 port1 0x330 port2 0x388 
pnp 1 1 os disable 
pnp 1 2 os enable port0 0x620 port1 0xa20 port2 0xe20 
pnp 1 3 os disable 
quit
-Mude a linha do /boot/defaults/loader.conf que diz:
userconfig_script_load="NO" 
-Para:
userconfig_script_load="YES" 
-Reinicie seu sistema
shutdown -r now 
-Depois de reiniciar digite os seguintes comandos:
cd /dev
sh ./MAKEDEV snd0 
-Reinicie seu sistema e sua placa AWE64 estara configurada

03- T1P5
+-+-+-+-

Como trocar o seu ip pelo IE4 ou superior:
obs.: nao foi testada em versoes anteriores, quanto ao netscape o processo 
deve ser parecido e provavelmente no Opera(meu browser preferido) tambem.
Vamos comecar, por primeiro temos que conseguir um end. de um proxy, algo 
extremamente facil, basta visitarmos a pagina
Proxy4All (http://proxys4all.cgi.net/) e pegar o endereco la.
Agora com o end., abra o IE e procure por "Opcoes da Internet", depois
em "Conexao". No caso do IE4, voce vera "Servidor Proxy", entao marque a
opcao que diz "Usar um servidor proxy", coloque o end. dele e sua porta e
pronto. Ja no IE5, voce tera que entrar "Configuracoes de Lan" e ai voce
vera "Servidor Proxy", marque a opcao que diz "Usar um servidor proxy",
coloque o end. dele e sua porta e esta acabado.Feito isso, quando voce
entrar em alguma pagina, ela nao pegara o seu ip e sim o do proxy, mas isso
nao significa que voce estara totalmente anonimo,afinal provavelmente seu ip
estara registrado em algum registro na maquina do proxy. 
[DuCkWaGe]

irc tips
--------

*1*
dando um lame takeover perfeito:

1- pegue op (duh!!!)
2- tire op de todos outros ops
3- de um ban *!*@* (/mode #canal +b *!*@*)
4- kicke todos usuarios (ops e usuarios comuns)
5- depois de ficar sozinho no canal de um mode +smilk 1 1
              (/mode #canal +smilk 1 1)
6- o canal eh todo seu

faq:

P: e se o canal for registrado e o chanserv devolve o op pro dono verdadeiro?

R: vc se fode e perde op
---------------------------------
*2*
como evitar lamers de falar no meu canal?

deixe seu canal em moderated (+m)
assim so quem tem voice vai poder falar no canal
pra dar voice: /mode #canal +v (ou -v) nick
---------------------------------
*3*
usando a telnet como cliente de irc

pra conectar vc deve saber ne
tanto linux como windows: telnet irc.xxxxxxx.com porta (mais comum :6667)
ai vc digita:
user (username nao eh nickname) (hostname) (servername) (realname)
ex: user ninguem localhost localhost ninguem
depois digita o nick:
nick nobody
os comandos sao na maioria iguais ao cliente de irc (/join,/mode,/nick, etc)
mas na telnet vc tira a /
nick eu
join #bl0w
mode #canal +b *!*@*

so + uma coisa
se por acaso vc receber algo do tipo:
ping :12345
ou ping :127.0.0.1
vc tem q responder assim:
pong :(o numero q vier)
pong :12345
ou pong :127.0.0.1
senao vc sera desconectado do servidor irc

ultima coisa:
se vc receber algo do tipo:
:nickname!username@xxxxx.com PRIVMSG seunick :_VERSION_
isso significa q o nick ae ta te mandando um ctcp pra saber q cliente c ta
usando
pra mandar de volta:
NOTICE nick :_VERSION Cliente q c ta usando
ex:
NOTICE nick :_VERSION Telnet 1.0

05- S1nGl3 Us3R b00t (Fr33BSD)
+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-

Bom, pra quem nao sabe, dar um boot como single user, 
siginifica ter praticamente acesso total ao sistema, 
sem precisar de nenhuma senha. Isto eh muito util quando 
se esquece a senha do root, ou caso aconteca algum problema 
que lhe impeca de entrar no sistema. Nao existe razao para 
desabilitar essa opçao, se ninguem alem de voce tenha 
acesso ao sistema, caso contrario, faca o seguinte para 
desabilitar o boot como single user:  
 
-Edite o arquivo /etc/ttys e altere a linha:

console    none          unknown   off   secure

para: 
console    none          unknown   off   insecure

-Depois, execute o comando: 
# kill 1 1

Pronto, agora so poderao ter acesso total ao sistema, 
fornecendo a senha do root. Pra quem nao sabe como dar 
boot com single user, eh so fazer o seguinte, digite
"boot -s" ou "boot -single". Eh isso ae, ate a proxima zine.

mirc proxy eh um addon pro mirc mto interessante
eh uma especie de firewall,vc vai servir de server pros outros
mas vai ver tudo q os outros fazem no irc (quem estiver conectado em vc)
pra loadear ele eh so digitar no mirc:
/load -rs addon\proxy.mrc

logico q aqui o diretorio onde ele ta eh no addon mas vc muda ae se o botar
num lugar diferente ne
pra conecta em vc basta o usuario digitar no mirc dele:
/server seuip:6999 irc.xxxxx.com.br

pra ver o help dele eh so digitar
/proxy help
