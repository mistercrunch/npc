2.1) SENTENCIAS DE CONTROL DE UN PROGRAMA.

Las sentencias de control son un grupo de palabras que indican las operaciones
mas comunes que se pueden realizar en C, tales como cual es la proxima
sentencia a ejecutar o cuantas veces evaluar una determinada expresion. El
numero de sentencias de control es reducido sin embargo con ellas se pueden
construir programas enormemente complejos. Estas sentencias las podemos reunir
en varios grupos.


2.2) SENTENCIAS DE SELECCION.

En C, el caracter ; sirve para separar las diferentes sentencias y las llaves
{ } sirven para agrupar declaraciones y sentencias en una sentencia compuesta
o bloque. Las sentencias que vienen a continuacion nos permiten elegir que
sentencia, simple o compuesta sera la siguiente en ejecutarse:


2.2.I) IF-ELSE.  Esta sentencia permite elegir si se va a ejecutar una
sentencia u otra en funcion del valor booleano de una expresion. Su sintaxis
es:

        if (expresion)
          sentencia_1;
        else
          sentencia_2;

Se evalua la expresion, y si es cierta (valor distinto de cero), se ejecutara
la primera sentencia, y en caso contrario la segunda sentencia. Aqui hay que
senalar que si se desean ejecutar varias sentencias en cada uno de los dos
casos, estas deben incluirse en llaves. Ver ejemplo 1 al final del tema.


2.2.II) ELSE-IF.  La combinacion la opcion ELSE con un nuevo IF permite la
realizacion de anidamientos que se pueden usar para hacer selecciones
multiples. El ejemplo numero 2 ilustra esta posibilidad.


2.2.III) SWITCH.  El caso de la seleccion multiple visto en el apartado
anterior es una construccion habitual en los programas por lo que todos los
lenguajes de programacion tienen instrucciones que la realizan de forma
sencilla. Aunque en el lenguaje C se puede usar ELSE-IF es bastante mas
adecuado el uso de SWITCH, cuyo formato es el siguiente:

        switch(expresion)
          {
            case expresion_1 : [sentencia_1;=20
            case expresion_2 : [sentencia_2;
            ...
            [default: sentencia_n;
          };

El switch evalua la expresion que debe ser necesariamente entera, el resultado
lo compara con cada uno de los casos y ejecutara aquel que coincida con el
valor hasta que encuentre una sentencia BREAK o CONTINUE. El ejercicio numero
3 resuelve el ejercicio igual al 2 pero utilizando Switch.



2.3) SENTENCIAS DE ITERACION.

Las sentencias de iteracion son las que permiten al programador ejecutar una
sentencia o grupo de sentencias un numero determinado de veces. Esto es lo que
en programacion se denomina bucle. El lenguaje C dispone de las siguientes
instrucciones para realizar este trabajo:


2.3.I) WHILE.  La instruccion WHILE  permite repetir una sentencia o grupo de
sentencias mientras una determinada expresion sea cierta. Su sintaxis es:

        while (expresion)
          sentencia

Es importante que en la sentencia o sentencias incluidas en la funcion WHILE
modifiquen la expresion que controla el final de la ejecucion del bucle, ya
que no ser asi se entraria en un bucle sin fin. El ejercicio numero 5 manda
escribir al ordenador diez veces un mensaje.


2.3.II) FOR.  La proposicion FOR es semejante a WHILE, aunque su sintaxis es
totalmente diferente:

        for (inicializacion ; condicion ; incremento)
          sentencia

La inicializacion permite dar un valor inicial a la variable de control de
bucle, mientras que la condicion determina cuando finalizara el bucle y con
incremento se indica como debe variar la variable de control cada vez que se
repite el bucle. En el ejercicio 8 vemos una forma de usar FOR.


2.3.III) DO-WHILE.  Las funciones anteriores primero evaluan la condicion y si
es cierta se ejecutan las sentencias, pero a veces lo que interesa en que las
sentencias se ejecuten al menos una vez, o sea, que se ejecutan en primer
lugar las sentencias y despues se evalua la expresion, si esta es cierta, se
ejecuta la sentencia de nuevo y asi sucesivamente. Su formato es:

        do
          sentencia
        while (expresion);

El ejercicio 9 muestra un ejemplo de uso de DO-WHILE. Realizar el ejercicio
10.



2.4) SENTENCIAS DE SALTO.

Los metodos de programacion estructurada sostienen que con las sentencias
vistas anteriormente se puede construir cualquier programa. No obstante en
determinadas circunstancias puede interesar al programador variar
incondicionalmente el flujo de la ejecucion del programa por lo que C dispone
de las siguientes instrucciones:

2.4.I) BREAK.

Esta sentencia permite salir sin condiciones de un ciclo for, while, do-while
y switch. Teniendo en cuenta que si estamos en bucles anidados se sale del
bucle mas interno. Tal y como vimos en el parrafo anterior el unico caso en
que esta justificado el uso de esta sentencia es para separar los diferentes
casos de una sentencia switch.
=20

2.4.II) CONTINUE.

Es una instruccion menos agresiva que break, y cuando se usa significa que el
control se pasa a la etapa de reinicializacion en un ciclo for. No tiene
ningun efecto en switch.


2.4.III) SALTOS Y ETIQUETAS.

La instruccion que permite hacer saltos incondicionales hacia otra parte del
programa senalado por una etiqueta es goto, es la instruccion mas aborrecida
por los amantes de la programacion estructurada. Nosotros no la usaremos en
ningun ejercicio este curso.



2.5) LAS FUNCIONES.
  
Una funcion consiste en un conjunto de datos e instrucciones que resuelven un
problema claramente definido.

Un programa esta formado por instrucciones que realizan las operaciones a lo
largo del mismo. Estas instrucciones se pueden agrupar en las llamadas
funciones, evitando la repeticion innecesaria de una parte del programa.

El funcionamiento de un programa compuesto por funciones es el siguiente:

 - El programa se ejecuta secuencialmente ?.

 - Detecta una funcion ?.

 - Ejecuta dicha funcion ?.

 - Continua secuencialmente con el programa donde lo dejo ?.


Una funcion esta compuesta por los siguientes elementos:

 - Cabecera: Es el nombre de la funcion seguida de los argumentos entre
             parentesis y separados por comas en caso de existir.
 - Declaracion de argumentos: Indicando los tipos de los mismos.
 - Declaracion de variables locales:=20
 - Sentencia compuesta: Codigo de funcion.
 - Valor devuelto.

Para ejecutar una funcion debemos escribir el nombre de la misma seguida por
los valores de los argumentos separados por comas y entre parentesis.

Todo programa escrito en C debe estar compuesto por funciones (una o mas). Una
dde estas funciones debe ser la funcion main. Los programas en C empiezan
siempre ejecutando esta funcion. En el ejercicio numero 11 vemos un ejemplo de
uso de las funciones, donde se utilizan variables globales validas para todo
el programa.

En el ejercicio numero 12 el programa calcula el area de un triangulo
utilizando una funcion donde se pasan valores de la funcion main a la funcion
que calcula el area, devolviendo esta ultima el valor calculado.
