Montando seu próprio FTP site no seu micro(r):
(mais um pedaço do meu livro)

Derneval R.R. Cunha

Instalar servidor ftp (ftp daemon) no seu micro é algo que pode ser interessante. Supondo-se que você tenha amigos ou colegas de trabalho com quem "troca" arquivos, facilita muita coisa. Claro, quando se tem uma boa ligação com a internet, pode-se usar o ICQ. Ou colocar o(s) arquivo(s) num site qualquer da WEB. Mas a troca de arquivos micro para micro via ftp é muito mais rápida do que via um site da web que pode estar lotado em determinadas horas do dia.

Observação: Na hora de divulgar, cuidado! Se você vai espalhar num chat que está usando seu micro para isso, pode não só atrair a ira do seu provedor, como também algum imbecil que vai tentar usar seu servidor ftp como porta de entrada.

Alguns bons softwares são:

FTP Serv-U 2.5d:

Altamente recomendável, tem um manual muito bom (em inglês) e está disponível para todas as plataformas Windows, do 3.11 ao NT. Só que é shareware e tem dois tipos de uso. Um que deixa você usar como se fosse o dono, mas por um período de tempo, tipo 30 dias. É preciso registrar (leia-se pagar a taxa) para continuar usando depois. A outra possibilidade é usar pelo tempo que quiser, mas a pessoa do outro lado só poderá fazer um número de operações de download/upload antes de travar sua sessão. site oficial: http://www.cat-soft.com/

War FTPd 1.65

(saindo a beta 1.70) É um dos mais populares e é gratuito. Muuuuito bom. Disponível para Windows 95/NT. Tem tanto modo simples como avançado de uso e várias características do Serv-u. Além disso, a segurança tá na ordem do dia. Funciona bem. Tem um tutorial passo a passo no http://www.acd.ucar.edu/computing/Guide96/internet/win95ftp.html O site oficial é http://www.jgaa.com/

Para usar um FTP Deamon (Servidor FTP) com segurança

Uma página com ferramentas e dicas de segurança, além de manual, está no ftp://ftp.cat-soft.com/add-ons/. Trata-se da página de ajuda do Serv-U, porém algumas dicas e ferramentas podiam servir para o War-ftp, que vale a pena lembrar, é gratuito.. Pensa aí com seus botões qual que é o melhor. É muuuuiito legal (e inseguro) colocar a disposição dos amigos tal tipo de serviço. Existe o problema da divulgação, já que a maioria da moçada não vai dispor de ip fixo, mas sim o chamado ip dinâmico, aleatório, fornecido pelo servidor internet através do qual a pessoa acessa a internet. É necessário divulgar esse número para a moçada poder conectar com o seu micro. O caso seria estar conectado via icq com a galera com quem você precisa fazer intercâmbio de arquivos. Aí é só usar o comando Add/Change current user ->View-Change my Details para descobrir qual o seu ip e mandar para todo mundo que está na sua lista. Essa é a maneira mais simples e menos sofisticada.

Claro, existem outras ferramentas para se fazer isso automaticamente. São os chamados IP-POSTERS. Pode-se colocar uma página na internet com o ip que está sendo usado. É só avisar seus amigos que de tal hora a tal hora, é só clicar na página. Existem ferramentas para isso, algumas boas, outras nem tanto. Vêem o seu ip e já editam um arquivo html que pode ir para a internet logo em seguida. Uma boa ferramenta gratuita está disponível no http://members.tripod.com/~krazy6284 . Outra está disponível no http://www.capital.net/~skibby. Também gratuita. Outra está no ftp://ftp.cat-soft.com/add-ons/whatmyip/ . Esse daí é via Prompt do DOS.

O WhatMyIP, requer um arquivo html comum de entrada, com a string de caracteres (ou sequência de letras) ### onde o ip tem que aparecer. Talvez não funcione com NT.

Última alternativa, precisou, digita:

netstat -n no prompt do DOS, anota o resultado e joga você mesmo na sua página.

Uso do Ftp server e a segurança básica:

Há outras dicas, como a questão de se fazer senhas para os usuários que vão acessar o servidor ftp. O serviço Anonymous é quando qualquer um pode acessar simplesmente digitando algo parecido como:

ftp://290549.324094.20954.2095 ou

ftp://ftp.servidorftpnoseumicro.lsjfalask.com (só para os manés entenderem a idéia, esse exemplo)

no seu Netscape ou no Explorer

A alternativa de usar senhas para impedir qualquer mané de entrar presume um planejamento, como colocar uma seções de:

Incoming - Para onde a pessoa pode copiar coisas dela para o seu micro

Pub - De onde tudo pode ser copiado

O acesso via senha daria possibilidade de se fazer "contas" ou "pastas" específicas para cada usuário cadastrado (isso pode ser uma dor de cabeça). O usuário acessaria então com um comando específico:

ftp://usuario:senhadousuario@ftp.servidoftpnoseumicro.com

É necessário também se precaver e não permitir que o servidor ftp permita a execução de programas (se você não entende do que estou falando, leia o manual do serv-U e se continua não entendo, repita a leitura até entender).

Conclusão:

Fazer um ftp site é um bom passatempo e ensina vários truques de segurança úteis para quando você for o Sysop ou Webmaster, mais tarde. É preciso muito cuidado com o que se está planejando fazer. Um ftp server no seu micro é a mesma coisa que deixar a porta dos fundos do seu computador aberta. A palavra chave é intercâmbio. Se você tem uma relação de cópia de arquivos com algum micro ou uma comunidade de micros depende de material que você mantém no seu micro, aí vale a pena se interessar em manter um ftp server on-line. Caso contrário, jogue seus arquivos em algum site gratuito na internet.
