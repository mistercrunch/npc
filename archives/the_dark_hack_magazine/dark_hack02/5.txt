1 Microsoft Windows 2000 - DoS en Servidor de Telnet
  ------------------------------------------------------
BugTraq ID: 1414
Remoto: Si
Fecha de publicacion: 30-06-2000
URL relevante:
http://www.securityfocus.com/bid/1414

El servidor de Windows 2000 viene con un servidor de telnet para proveer
acceso remoto mediante consola. Un atacante remoto puede mandar una corriente
de ceros binarios al servidor, causando que este falle. Si el automatic restart esta habilitado, numerosos ataques provocaran que el servidor de telnet se reinicie hasta que el contador de fallas/reinicios se exceda, punto
en el cual el servicio de deshabilita.

2 Microsoft Windows 2000 - Sobrecarga de CPU remota
  -------------------------------------------------
BugTraq ID: 1415
Remoto: Si
Fecha de publicacion: 30-06-2000
URL relevante:
http://www.securityfocus.com/bid/1415


 Mandando una corriente de ceros binarios a cualquiera de una serie de
puertos de Windows 2000 puede provocar la utilizacion del CPU al 100%.
 Los puertos que fueron encontrados vulnerables son los puertos TCP 7, 9,
21, 23, 7778 y los puertos UDP 53, 67, 68, 135, 137, 500, 1812, 1813, 2535,
3456.

4 Microsoft FrontPage Server Extensions 2000 - Divulgacion de Path 
  ----------------------------------------------------------------

BugTraq ID: 1433
Remoto: Si
Fecha de publicacion: 04-07-2000
URL relevante:
http://www.securityfocus.com/bid/1433


Las Microsoft Frontpage 2000 Server Extensions divulgaran el path fisico
completo del directorio raiz virtual de la web si un usuario remoto solicita
un archivo no-existente en el directorio /_vti_bin/shtml.dll. Por ejemplo,
realizando un request de:
http://target/_vti_bin/shtml.dll/no_existo.html   provocara un mensaje de 
error  diciendo "Cannot open "C:\path_fisico\no_existo.html": no such file or
folder"

6 Novell BorderManager - Vulnerabilidad de personificacion de usuario
  -------------------------------------------------------------------

BugTraq ID: 1440
Remoto: Si
Fecha de Publicacion: 2000-07-07
URL Relevante:
http://www.securityfocus.com/bid/1440


 El Novell BorderManager implementa una aplicacion llamada ClientTrust 
para manejar procedimientos de login. ClientTrust escucha en el puerto
3024 de la maquina del usuario para pedidos de autentificacion del 
BorderManager siempre que el usuario trata de acceder a la web. Este 
mecanismo hace factible que un usuario no autorizado se camufle como un 
usuario autorizado debido al hecho  que este no verifica el origen del 
pedido de acceso.
 Esto podria ser llevado a cabo a traves de redireccionamientos de puertos 
en el puerto 3024. Primero, el atacante necesitaria tomar una maquina en uso
de la persona que quiere personificar (la victima). Segundo, tendria que 
configurar port forwarding o reenvio de puerto en su propio sistema, para 
que cualquier pedido hecho al puerto 3024 sea reenviado a la maquina de la
victima. Entonces, hace su pedido o request. El servidor BorderManager 
permitira el pedido.
 Esto puede llevar a dos riesgos separados: Acceso a contenido no autorizado, 
y ataque a la persona accediendo a websites cuestionables (pornografia, 
sectas, etc.) usando la personalidad de la victima.
 OJO. La direccion IP correcta de la maquina del atacante queda loggeada.

8 Microsoft Excel 97 / 2000 - Vulnerabilidad de  Register.ID
  ----------------------------------------------------------
BugTraq ID: 1451
Remoto: Si
Fecha de Publicacion: 11-07-2000
URL relevante:
http://www.securityfocus.com/bid/1451


 Gracias a la funcion REGISTER.ID en Microsoft Excel 97 / 2000, es posible 
la ejecucion remota de codigo nativo contenido en un archivo DLL bajo ciertas
condiciones.
 La funcion REGISTER.ID debe referirse a un archivo DLL especialmente creado
que contenga el codigo arbitrario que sera ejecutado y este debe residir en
el disco rigido local o en un UNC compartido. Si un usuario abre un archivo
.xls que contenga esta funcion, el codigo especificado sera corrido con 
el nivel de privilegio del usuario.
 Excel y otros archivos de MS Office pueden ser enviados y abiertos 
automaticamente via navegadores y clientes de mail compatibles con HTML. En 
la mayoria de los casos, no se le dara ningun aviso al usuario antes de 
que Excel sea iniciado, y tampoco ninguna alerta antes que el codigo sea
ejecutado.

 
9 NAI VirusScan/NetShield - Vulnerabilidad de Verificacion de AutoUpgrade
  -------------------------------------------------------------------------
BugTraq ID: 1458
Remoto: Si
Fecha de Publicacion: 11-07-2000
URL Relevante:
http://www.securityfocus.com/bid/1458


 El Netshield y el VirusScan 4.5 no verifican el contenido de un ejecutable
corrido por la la funcion de AutoUpgrade. Una clave del registro controla
en que directorio residira el programa de upgrade, SETUP.EXE. Bajo la 
instalacion default este directorio no esta asegurado, permitiendo el 
reemplazo del SETUP.EXE, el que va a ser corrido con privilegios de 
administrador local. Es mas, si la clave del registro puede ser modificada,
cualquier programa llamado SETUP.EXE en un directorio con el formato correcto
(los archivos PKGDESC.INI y SETUP.ISS deben estar en el mismo directorio) 
puede ser corrido junto al AutoUpgrade.
