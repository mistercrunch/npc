Configuracion y Ejecucion de un Servidor Apache
	Por MaGuRa

Bueno en este articulo voya explicar como se configura el servidor Web Apache
para Linux, doy por echo ke ya lo tines instalado, compilado y demas movidas de
ese tipo. Bueno paso a explicar la configuracion del server:

Los ficheros desde donde se puede configurar el Apache son los siguientes:
 1. PREFIJO/etc/httpd.conf
 2. PREFIJO/etc/srm.conf
 3. PREFIJO/etc/access.conf

Bueno solo necesitamos un editor de texto, el que os mole eso da igual.
	
	1.Bueno vamos a empezar a configurar el fichero httpd.conf

En este fichero se encuentra la configuracion general del servidor. Tiene
mogollon de opciones pero la mayoria bienen configuradas por defecto (lo ke
no kierecer ke no las toques). Algo super importante para ke corra bien el
servidor es la directriz "SERVERTYPE", esta puede tener dos valores: 

----> standalone con esta opcion el servidor correra independiente
quedando a la escucha de un puerto TCP/IP. Esta es la configuracion por
defecto del servidor y la ke todo el mundo suele utilizar, con esta opcion el
servidor se clonara para poder atender varias peticiones a la vez quedando
siempre el servidor principal a la escucha por si llegan mas peticiones, a se
me olvidava las peticones maximas tambien se pueden definir en este archivo
para ke no se te llene de peña y vaya super lento

-----> inetd con esta opcion lo ke hacemos es integrar el servidor en el
superdemonio inetd. En este caso sera este superdemonio el ke se escuentre a
la escucha y cuando reciva una peticion lanzara Apache. Para hacer esto se
tiene ke configurar el archivo /etc/inetd.conf, donde encontraremos entrada
para el httpd. Generalmente esta dirigida al puerto 80. Para ejecutar Apache
en este modo habar ke hacer las siguientes modificaciones:

--------> fichero /etc/inetd.conf Hay ponemos lo siguiente" httpd stream tcp
nowait httpd /usr/sbin/httpd -f /etc/httpd/conf/httpd.conf "
* esto es suponiendo ke el +x de httpd esta en /usr/sbin

--------> fichero /etc/services en este habria ke poner lo siguiente " httpd
80/tcp httpd "
Bueno tas estas modificacioncillas tan solo hay ke reiniciar el daemon inetd
(pa el ke no sabe komo reiniciar el demonio "kill -HUP <pid de initd>")

	2. Continuamos con el srm.conf

Este fichero contiene las directrices de configuracion del sistema, se utiliza
para indicarle al server que recursos a que recursos se pueden acceder a
traves del sitio Web y como lo ha de hacer. Bueno en este archivo se indica lo
ke el servidor va utilizar iconos,idimas a soportar,etc.

Creo ke lo mas importante de este fichero es es el arbol de directorios ke
mostrara el servidor Web, asi como el nombre del fichero ke por defecto sale
al acceder al directorio. El valor del path donde comienza el arbol de
directorios del servidor se define en la directriz DocumentRoot.
Para configurar el fichero indice se utiliza la directriz DirectoryIndex.Por
defecto tomara los ficheros index.html o index.htm. Esto es importante ya ke
si no se pone lo de index.h* saldra el listado de ficheros de la web si ke
salga ninguna Web.




	3.Finalmente el access.conf

Como indica el nombre del fichero es obio ke este fichero sirve para determinar
el acceso de los usuarios hacia los permisos de los objetos ke se encuentran en
la/s Web/s. Se debe de modificar la directriz (claro ke si kieres ke te cambie
to dios los permisos de los objetos de la web no hace falta ke toques na XD)
<Directory></Directory>, enetre la cual encontramos una serie d definiciones.
en primer lugar lo ke debemos de modificar es el path colocoando lo ke hayamos
puesto anteriormente en la directriz DocumentRoot del fichero srm.conf. Bueno
entoces con esto ke konseguimos pos ke el servidor se quede en un modo en el
que acceda a este directorio y no sea necesareo efectuar ningun tipo de
modificacion.

-----------------------------EJECUCION----------------------------------


Si utilizamos el standalone tenemos ke ejecutar lo siguiente para poder correr
el servidor
$/bin/apachectl start

Si estamos conectados a una red nuestra direccion de pagina sera
http://nuestra-ip, si hemos utilizado un puerto sistinto al 80 sera
http://ip:puerto. Si no estamos conectados sera http://localhost
para para el servidor podremos $/bin/apachectl stop.
