# 2.0 Hardware


 El hardware es algo fundamental para nuestra red, en especial el de nuestro servidor
 porque si llega a estar dañado o tiene algun problema este no afecta solo al servidor
 si no que le afecta a la red entera.
 Otra de las cosas fundamentales es el cableado que conecta a todas las maquinas, 
 porque si tiene cortes o quebraduras en cualquier parte causa deconexiones y mal
 funcionamiento de ese pc en la red, hay veces en que uno piensa que el problema esta
 en la configuración y resulta que era el cable que estaba maltratado por dentro.
 En este texto voy a hablar basandome siempre en el cable UTP (unshielded twisted
 pair, que significa Par trensado sin blindaje), a que estos permiten mayores 
 velocidades de transmición y mayores distancias.
 Identificamos a los cables UTP por la ficha RJ45 que se encuentra en sus puntas.
 La conexión del cable seria la siguiente :

 Del Pin 1 al Pin 3
 Del Pin 2 al Pin 2
 Del Pin 3 al Pin 1
 Del Pin 4 al Pin 4
 Del Pin 5 al Pin 5
 Del Pin 6 al Pin 2
 Del Pin 7 al Pin 7
 Del Pin 8 al Pin 8

 (Espero que se entienda, que intente hacer el grafico pero se me lio)

 Para conectar las fichas RJ45 necesitamos una pinza crimpeadora.
 No sean brutos al conectar los cables, porque podrian romper las RJ45 o conectar mal
 algun cable y despues eso nos va a traer dolores de cabeza.

# 2.1 Placas de Red
 

 ¿Para que queremos esto? Para conectar nuestras PC´S con las demas de la red, esto
 seria nuestro enlase con las demas PC´S de la red y obviamente con el servidor.
 
 La placa que nosotros vamos a usar es la Ethernet, las mas usadas, baratas y muy 
 faciles de instalar y configurar, estas placas traen ficha para conectar la RJ45 que
 son las que usaremos en nuestro cableado.
 Estas placas se denominan como 10 Base T y tienen una velocidad de transmición de 
 10 Mbits, si queremos placas mas rapidas podremos encontrar la 100 Base T y con una
 velocidad de 100 Mbits una gran diferencia comparada con la primera pero esta tiene
 una desventaja y es que vale un poco mas caro.
 Cada una de estas placas tiene un número asignado de seis bytes denominado MAC 
 address y esto es como una IP, cada una tiene un número asignado y NUNCA se repiten.


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 2.2 Hub

 
 El Hub es una especie de consentrador que se encarga de distribuir la información a
 toda la red.
 El Hub es un aparato con ocho bocas en las cuales se conectan los cables de nuestra
 red, hay distintos tipos de Hub´s con distintas cantidades de bocas y distintas 
 velocidades de transmición .
 Los Hub´s funcionan con redes Ethernet y con cableado UTP.
 Si nuestras placas de red son 10 Base T tendriamos que elejir un Hub de 10 Mbits.

# 3.0 S.O

 
 El Sistema Operativo es fundamental para nuestra red, no es cosa de poner cualquier
 sistema, si no que necesitamos un sistema capaz de compartir recursos y usar los 
 mismos protocolos para comunicarse con las demas PC´S de la red.
 Los protocolos que utilizaremos son el NetBIOS y el NetBUI y algunas de las
 plataformas que podemos usar son :

 Windows 3.11 / 95 / 98 / Nt / XP

 OS/2

 Mac Os 9x.

 Unix

 Linux (slackware, redhat, debian, mandrake, gentoo, openlinux, conectiva, SuSe, etc)

 BSD (Free, Net, Open)

 Solaris

 IRIX

 NetWare

 Para un usuario que quiere montar una red en su casa para uso local y sin mucha 
 experiencia, aconsejaria Windows.
 Pero si hablamos de una red mas grande aconsejaria un Slackware, pero no todos los 
 usuarios lo saben manejar, asi que podriamos poner unas maquinas en Linux y otras en
 Windows y con la herramienta Samba hacer que se puedan comunicar sin problemas.
 Todas las PC´S deberan usar el protocolo TCP/IP, NetBIOS y NetBUI.

 Si queremos montar una red de menos de quince Pc´S, sin mucha seguridad ni 
 organización de datos usaremos peer to peer (de igual a igual).

 Lo que mas debemos cuidar es el servidor hay que administrar bien su software y 
 hardware para un mayor rendimiento, porque si se nos llega a caer el servidor se nos
 cae la red entera.

# 4.0 Internet


 Ahora llega la hora de conectarnos a Internet, tenemos que conectar todas las PC´S de
 nuestra red por nuestro ISP.
 Lo primero que haremos sera instalar el protocolo TCP\IP en todas las PC´S de nuestra
 red.
 Cuando configuramos y instalamos este protocolo nos pedira que asignemos una IP a 
 cada PC, cada PC tendra una IP única que ninguna otra podra tenerla.
 Para eso vamos a asignar una ip  que sigan todas el mismo rango, EJ. :

 127.0.0.1 <- Server
 127.0.0.2 <- Maquina 1
 127.0.0.3 <- Maquina 2
 127.0.0.4 <- Maquina 3

 etc, etc...

 Una vez que instalamos y asignamos la IP de cada maquina estamos listos para conectar
 con nuestro ISP, pero el problema es que nuestro ISP nos asigna una sola dirección de
 IP que va destinada a la maquina que esta conectada al router, en este caso, el 
 servidor.
 Ahora, si no tenemos una IP fija deberemos definir dos direcciones de IP, la primera
 es la que asignamos para nuestra red, y la segunda es la que nos asigna el servidor.
 Pero para eso necesitamos dos adaptadores de red diferentes, tendremos una placa de
 red con la ip que asignamos a nuestra red y otra que puede ser un router o otra placa
 de red con la IP que nos asigna el servidor.
 Ahora esta PC sera nuestro gateway (enlase) entre nuestra red y nuestro ISP, podemos
 verlo en el siguiente esquema :

   ___		    ____	     ____	      ____
  |   |		   |	|	    |    |	     |    |

 (aunque no parezca, esos dibujitos son computadoras)

 Seria un enlase asi : Red <=> Servidor <=> ISP .

# 4.1 Proxys

 
 Ahora, nuestro ISP solo nos asigo una dirección de red y el lo ve como una sola PC.
 pero, como hacemos para que todas nuestras pc´s se puedan comunicar usando esta 
 conexión y poder controlar y asegurarce de que todas las transmiciones se reciban
 correctamente?, muy facil, con un proxy.

 ¿Qué es un Proxy? Un programa que administra todo el el trafico que entra y sale de 
 nuestra red. Este tambien se encarga de filtrar los paquetes malignos que puedan 
 causar problemas en nuestra red y protegernos de ataques.
 El proxy tambien nos puede servir para restringir ciertas funciones de nuestras PC´S
 para que un usuario normal no pueda hacer ciertas cosas.
 Tambien contiene una carpeta en la que va guardando los sitios mas visitados para que
 si otros quieren entrar a ese sitio el proxy lo tendra en su caché y asi reducir el
 trafico de nuetra red.

# 5.0 OSI

 El modelo OSI (Open Systems Interconections, Conexión entre Sistemas Abiertos),
 nace en los años 70's y su objetivo era poder establecer conexiones entre 
 maquinas qué usen distintas plataformas, conexiones, arquitecturas, etc.

 Para esto el ISO (Organización Internacional de Normalización) ideo el modelo 
 OSI qué consta de 7 capas o niveles, cada una de ellas cumple una función 
 especifica y se encarga de negociar los protocolos qué se usaran entre los 
 sistemas.

 Los siete niveles son :

 	
 		1) Fisico - Hardware

 		2) Enlace de Datos

 		3) Red

 		4) Transporte

 		5) Sesión

 		6) Presentación

 		7) Aplicación

 
 Cada nivel depende del que le sigue, osea, el qué tiene por antemano, pj: 
 El nivel de Enlase de Datos no podria funcionar sin el nivel Fisico.
 
 Casi la totalidad de las redes usan el modelo OSI.

 A continuación voy a describir las 'funciones' básicas de cada nivel.


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 5.1 Nivel: Fisico


 Bueno, este es el mas importante porque de el dependen los demas, este hace
 referencia a todo el hardware de nuestra pc y a todas las conexiones de nuestra 
 red y tambien a todo medio de transmición, como hubs, switchs, placas ethernet etc.

 Tambien se ocupa de las operaciones a niveles BIT, pj: que 1 bit se reciba a 1 bit.


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 5.2 Nivel: Enlase de Datos


 Este se encarga de mantener las conexiones de los usuarios.

 Controla los paquetes qué circulan por la red y se encarga de hacer qué lleguen
 bien. 

 Tambien controla y corrige los errores qué puedan llegarce a producir y 
 los errores de nivel fisico en la transmición de datos ; para eso divide los datos
 en tramas.

 Tambien se encarga de controlar de qué no se produscan coliciones, para eso asigna
 tiempos diferentes a cada paquete.

 Tambien controla la sincronización de datos.

 Si hay comunicación bidireccional esté nivel se ocupa de ello.

 Este nivel empieza a funcionar cuando se establecio la conexión fisica.


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 5.3 Nivel: Red


 Es el encargado de llevar los datos hacia su objetivo elejiendo el camino más 
 satifactorio de la red. 

 Se encarga de establecer las conexiones entre maquina y maquina qué NO esten en el
 mismo medio fisico (osea que podria conectar dos maquinas atravez de inet..).

 Tambien se encarga de controlar el trafico de la red y establecer conexiones entre
 maquina y maquina con distinto hardware.
 
 Este nivel usa el protocolo IP y IPX para interconectar redes y tambien usa los 
 protocolos Frame Relay, ATM, X25 ...


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 5.4 Nivel: Transporte


 Este nivel es el qué se encarga de qué nuestros paquetes lleguen a su objetivo
 sin problemas y correctamente. El nivel Transporte usa el protocolo TCP, qué es el
 qué asegura el envio de nuestros datos.
 
 Es capaz de realizar y mandar datos usando el protocolo UDP cómo el TCP. 

 Tambien se ocupa de separar los paquetes y formar datagramas qué despues seran
 ensamblados en la maquina qué los recibio, esto lo hace para aprovechar mejor la
 conexión.

 Puede multiplexar varias conexiones sobre una misma conexión de red o distribuir una
 conexión de transporte en varias conexiones de red. 

 Es capaz de establecer conexiones de maquina a maquina directamente (osea sin 
 intervenciones).

 Bueno, hasta aca vimos los primeros cuatro niveles, qué se denominan "Servicios de
 transporte". Los otros tres qué le siguen son niveles de "proceso".


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@

# 5.5 Nivel: Sesión


 Este nivel es el encargado de organizar las conexiones de la red, se encarga de
 establecer y cerrar conexiones y qué los niveles superiores puedan enviar los datos
 sin problemas.

 Un ejemplo tipico puede ser el de los programas de intercambio cómo el kazaa, imesh,
 emule, etc, que albergan a mas de un usuario.

 Se ocupa de que un usuario pueda establecer multiples seciones a la vez.

 Organiza y sincroniza el diálogo entre los usuarios.

 Tambien controla los tiempos de la comunicación y algunas funciones cómo seguridad, 
 administración de contraseñas y sistema.


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 5.6 Nivel: Presentación


 El nivel Presentación se encarga de organizar y preparar datos qué luego seran 
 enviados y/o utilizados por alguna aplicación.

 Traduce los datos para qué despues puedan ser leidos por alguna aplicación y/o 
 usuario.

 Tambien se usa para encriptar o desencriptar datos ó comprimir/descomprimir datos 
 para alivianar el trafico de las conexiones.

 Capaz de comprimir los datos.


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 5.7 Nivel: Aplicación


 Septimo y último nivel, usado por nuestro software de comunicaciones, cómo clientes
 de FTP, controla y organiza los programas de comunicación.

 Proporciona comunicaciones entre dos aplicaciones, de PC a PC.

 Es lo qué hace que el usuario tenga acceso al entorno OSI mediante software (ap-
 licaciones).

 Usa protocolos como FTP, HTTP, SMTP ... 

 Osea, se encarga de proporcionar los servicios al usuario.


-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@-@


# 5.8 Resumen OSI


 Resumen de los siete niveles OSI :

 1° Fisico : Hace referencia al Hardware y transmiciones de Bits.

 2° Enlace de datos : Congestionamiento. Control de errores. 

 3° Red : Enrutamiento de paquetes. Compatibilidad entre redes & hardware.

 4° Transporte : Asegurar el envio de datos.

 5° Sesion : Establecer comunicación entre usuarios. Administración de contraseñas.

 6° Presentación : Traducir los datos para poder ser entendidos. Encriptación y compreción.

 7° Aplicación : Software para la comunicación.

# 6.0 TCP / IP  Rápido !


 Ahora vamos a tratar de comprender en pocas palabras el protocolo TCP/IP .

 Qué es ?  Digamos, es un conjunto de 'codigos' que nos permite la comunicación entre 
 ordenadores. Esos codigos, son protocolos, osea, estandares de comunicación.

 Cómo funciona ?  Convierte nuestra información al lenguaje de internet, osea, 
 crea un paquete y se asegura que sea enviado y recibido en su totalidad y en el orden
 correcto. 
 Mientras que el IP (más abajo explico esto) le pone remitente y destinatario a cada 
 paquete y se encarga de enviarlos.

 IP : (Internet Protocol) Es una dirección que nos asigna nuestro ISP para identificarnos
 ante la red, cada dirección es ÚNICA, sería como la calle donde vivimos. Es la manera de 
 ubicarnos. Una Ip consta de 12 caracteres , ejemplo, 200.230.198.132 . Van desde :
 127.0.0.1 hasta 255.255.255.255.

 Hay 3 tipos de direcciones: Clase A , B Y C .


 Bueno básicamente es eso, esto es extremadamente resumido, lo ampliaré en futuras versiones
 disculpen.

# 6.1 UDP Rápido ! 


 El UDP (User Datagram Protocol) agrupa todos los datos en un datagrama y sin comprobar
 a quien será mandado, envía ese datagrama, no es muy fiable, pero es un protocolo más.
 Este protocolo no realiza ninguna conexión, envía el datagrama al destino . Si llega

# 6.2 ICMP - PING


 Cuando se abla de ICMP se habla de ping, TCP/IP y paquetes, 
 a continuacion se explicare todo esto.
 
 El ping funciona de una manera muy simple: envia un paquete y espera una respuesta.
 Este utiliza un protocolo incluido dentro de TCP/IP, el ICMP 
 (Internet Control Message Potocol, Protocolo de Internet para  Mensajes de Control). 

 El protocolo ICMP se hizo para detectar errores en redes, tiene muchas funciones
 Una de ellas es la llamada "ICMP_ECHO" cuando un sistema recibe esto devuelve 
 "ICMP_ECHO_REPLY", Esto sirve para saber si el host esta activo.

 Siguiendo con el Ping, hay varios tipos de respuestas que te dan informacion de un host. 
 En los casos anteriores (ICMP ECHO), son los tipos 0 y 8, pero hay otros tipos de paquetes
 ICMP, y sus repuestas nos permiten averiguar el horario local del sistema
 mascara de subred, el SO que usa, etc, etc..

 Casi siempre se utilizan herramientas para realizar lo que se conoce 
 como "barrido de ping", que permite scanear redes completas, o a intervalos de red
 con las que hay que evitar *direcciones de broadcasting* (imaginarce lo q pasaria X'D). 

 Tambien hay que evitar las direcciones IP no redireccionables, las que se utilizan en 
 redes LAN (Local Area Network, x si no sabeis xD) . 
 
 Muchos ISP's muestran las direcciones de sus redes internas al hacer 
 Trace Route (Trazado de Ruta).

 Sin dudas, las herramientas mas avanzadas de este tipo son aquellas basadas 
 en sistemas Unix, que hay de a montones para GNU/LiNUX. 

 A comparacion de las de Windows (q estan basadas en interfaces graficas), las de
 UNIX permiten su utilizacion dentro de scripts, lo que permite la combinacion 
 con otras herramientas, demas esta decir que las implementaciones TCP/IP son mas completas en sistemas UNiX.

 Con el paso del tiempo se le fueron dando "malos" usos al ping.

 Uno de estos es el famoso "ping de la muerte", consiste en aumentar el tamaño 
 del paquete enviado por ping, lo que aparentemente no era posible
 pero se fragmentaba el paquete y, al ser reconstruido en el destino
 causa Buffer Overflow (desbordamiento de buffer) lo que provoca cuelges o desconexiones.

 Otro es el ataque "Smurf", consiste en enviar ping a *direcciones de broadcast* 
 modificando la direccion de origen, lo que causa una negacion de servicio al recibir respuestas de almenos 255 host.
 
 Para evitar estos "ataques" se recomienda un firewall.

# 7.0 Seguridad


 Si tenemos una red peer to peer que esta en un cybercafé para jugar a los juegos, no
 creo que necesitemos mucha seguridad, pero podriamos restringir ciertos accesos y 
 controlarla muy bien (que a determinada hora se bloquee, P.E) podriamos usar el 
 CyberControl, es muy facil de usar y es de tipo NAT (cliente/servidor), instalamos el
 server en la maquina de nuestro mostrador y los clientes en las maquinas de nuestra 
 red.
 Así podremos controlar a la perfección los horarios, o que no se pueda acceder a 
 ciertas partes de la PC.

 Pero si tenemos una WAN  la seguridad es una de las cosas mas importantes.
 Para eso vamos a necesitar las siguientes cosas :

 # Un buen proxy. (que tenga filtrado de paquetes y conexiones)
 
 # Un buen antivirus. (Los virus son las peores amenzas para nuestra red, ya que las
 alentan, borran cosas, y algunos se distribuyen por toda la red. Buscaremos un 
 anti-virus que no nos ponga lentas todas las maquinas, como pude ser el PC-Cillin ó
 el F-Secure)

 # Monitoreo de la red. (Para detectar bugs en el hardware y el software. Un exelente
 programa es el WhatsUp GOLD, que hace un mapeo de toda la red e  informa el estado de
 los hubs, routers y cada puesto, si detecta alguna anormalia se comunica con el 
 administrador (via telefono, mail, peger)).

 # Establecer Contraseñas. (Establecer contraseñas si las maquinas comparten algun 
 recurso)

 # Firewall . (Si el proxy nos queda corto, instalemos un firewall para controlar los
 accesos a las maquinas y el filtrado de paquetes.) 

 # Testeos. (Podremos testear si la red es segura con algun programa como puede ser el
 Languard de Gfi, lo encontramos en www.languard.com)

 # BackUps . (Si algun virus o por algun motivo se pierde algun archivo siempre es 
 bueno tener hecho un backup de todos nuestros archivos y poder restaurar todo 
rapidamente)

# 8.1 Linux 
 

 Si usamos linux, podemos filtrar los paquetes indeseados con las IPTABLEs de 
 la siguiente manera : 
 
 iptables -D INPUT -p icmp DROP

 De esta manera bloqueamos todos los paquetes ICMP, y no podran hacernos PING´S.
 Para bloquear otro protocolo se cambia la palabra 'ICMP' por el protocolo que 
 queremos bloquear, ej, udp, tcp, etc.

                        . . : Red Con Linux : . .

 Bueno, si vamos a montar una red en linux tendremos qué "instalar" la placa de red (
 no voy a decir cómo configurar una placa de red.. buscá una howto..).
 
 Una vez echo esto.... :


 Server -> 4.4.1
 PC1    -> 4.4.2
 PC2    -> 4.4.3

 (Obviamente las ips son inventadas XD)


 Bien, comenzaremos a levantar las maquinas de la red, para esto :

 ifconfig eth0 4.4.1 up

 Debemos hacer esto en cada pc y cada pc con su ip.

 Despues editemos el archivo /etc/hosts y ponemos :

 4.4.1 SERVER
 4.4.2 PC1
 4.4.3 PC2

 Así no nos tendremos que liar cuando queramos hacer referencia a una maquina de la 
 red.

 Para ver si nuestras maquinas quedaron conectadas, hacemos ping :

 $ ping 4.4.2

 .....


 Podemos comunicarnos con los demas usuarios de la red usando el comando "talk", Ej:

 talk PC2

 Si PC2 quiere haceptar el chat con PC1, basta con poner :

 talk PC1

 
 Otra de las cosas qué vale destacar es que cuando reiniciemos, las configuraciones de
 la red se perderan, así que podemos editar el archivo rc.local y agregar las lines de
 ifconfig.

# 9.0 Samba  


 Samba es un software para poder trabajar con otras plataformas, el caso tipico es el
 de Windows Y Linux, por si queremos un servidor en Linux y las maquinas en Windows o
 combinar, algunas maquinas en Windows y otras en Linux.
 Lo que hace es crear un directorio en una de las maquinas y que se pueda acceder 
 desde cualquier maquina y asi compartir recursos, impresoras, y poder realizar 
 tranferencias de datos entre maquina y maquina.

 Esta herramienta la podemos encontrar en : www.samba.org .


		--------------- CONFIGURACIÓN DE SAMBA --------------------

 Editamos el archivo smb.conf y ponemos lo siguiente :

 [global]
 workgroup = b-04 #donde dice b-04 va el nombre de la red ;P
