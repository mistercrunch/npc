|===========~ 1- ¿ Qué es una red WAN ?. ~|=================================
|===========================================================================

Una red WAN ( Wide Area-Network ) o red de area extensa. Como todas las redes,
el objetivo de esta no es otro que el de poder establecer una conexión estable
entre varios equipos situados en diversos puntos para poder utilizar aplicaciones 
comunes y poder recurrir a datos o recursos de los sistemas que la forman.
Los equipos que forman esta red son llamados tambien Hosts, estos hosts están 
comunicados por una subred ( subred de comunicacion ).
 
-----------------------------------------------------------------------------
.==========================================================================.
|===========~ 2- Subred de comunicación. ~|=================================
|===========================================================================
 
La subred esta constituida por por elementos de conmutación y circuitos.
La función de los circuitos es la de mover datos entre los equipos. Los
elementos de comunicación son unas máquinas que sirven para enrutar la
información ya que conectan varias líneas de transmisión.

-----------------------------------------------------------------------------
.==========================================================================.
|===========~ 3- Tipos de redes WAN. ~|=====================================
|===========================================================================

-Conmutadas por mensaje:

Consiste en un equipo encargado de permitir o denegar el acceso a la red a las terminales que
lo soliciten. La aceptación o denegación del acceso se realiza después de examinar la cabezera 
del mensaje.


-Conmutadas por circuitos:

En estas redes, la conexión se realiza mediante una llamada. Una vez establecida la conexión
ya se dispone acceso a todos los datos de la red.


-Conmunatadas por paquetes:

Los datos se fragmentan en pequeños paquetes que se insertan dentro de informaciones
del protocolo, y van recorriendo la red como unidades independientes.


-Red orientada a la conexión:

Solo hay un camino para la transferencia de información. De esta manera no habra transferencia de 
datos por otro camino simultaneo.

-Red no orientada a la conexión:

Hay varios caminos de transferencia de datos, los datos se fragmentan y siguen distintas vias para
llegar al destino. ( Datagramas )
 
-----------------------------------------------------------------------------
.==========================================================================.
|===========~ 4- Topologías. ~|=============================================
|===========================================================================

-Topología de bus


Se conectan las estaciones con un solo cable, en forma lineal. Es muy fácil de configurar y 
la forma más sencilla de conectar los equipos. Lo malo está en que si se daña el cable, toda
la red dejaría de funcionar.


-Topología de anillo


En esta topología, los datos pasan por todas las estaciones y en un sentido único. Utilizan
el método de transmisión por token ( testigo ) que no es mas que una serie de datos que dan luz
verde al envio de información. Uno de los problemas de esta red radica en que si falla alguna 
estación, la red podria interrumpirse, aunque también se podria aislar y permitir asi el funcio-
namiento de la red.


-Topología en estrella


Es una estructura radial, las estaciones se conectan a un hub central. La ventaja es que si
falla algún equipo la red continua funcionando. La desventaja tal vez sea que el terminal 
central necesite mucho cableado y mucha seguridad.


-Topología en árbol


La red tiene un nodo en la " raiz " y de hay se van expandiendo hacia afuera. Esto permite
conseguir una red muy amplia.
 
-----------------------------------------------------------------------------
.==========================================================================.
|===========~ 5- Redes públicas,privadas, líneas analógicas y Digitales. ~|=
|===========================================================================

-Redes públicas

Son las redes que ofrecen recursos a los usuarios pertenecientes a compañías y que el usuario debe
pagar para su utilización.

-Redes privadas

Es una red que está totalmente en manos de una empresa para su utilización propia. Esto no impide
que parte de los servicios de esta red se ofrezcan a redes públicas. Puede que estas redes tengan
un costo muy elevado, pero también dota a la empresa de una seguridad mayor y privacidad.

-Líneas analógicas

Son las lineas de voz a las cuales se conecta el teléfono. En teoría no se podrian transferir
datos digitales de por si, pero se pueden modular utilizando un módem.

-Líneas digitales

Son líneas que tan solo se encargan del tráfico de datos, alcanzan mayor velocidad de transmisión
que las líneas analógicas y no necesitan modular la señal.

 
-----------------------------------------------------------------------------
.==========================================================================.
|===========~ 6- Componentes físicos de una red. ~|=========================
|===========================================================================

-Línea de comunicación
Todos los componentes físicos para conectar las estaciones con el fin de conseguir una conexión
estable para recibir y enviar datos.

	

-Hilos de transmisión
Los elementos que forman el circuito de la red. A estos hilos se les llama también pares.

 
-----------------------------------------------------------------------------
.==========================================================================.
|===========~ 7- Líneas de conmutación. ~|==================================
|===========================================================================

-Líneas Conmutadas 

Se establece un código para poder interconectar dos extremos.

-Líneas de punto a punto

Conectan dos terminales directamente.

-Líneas de varios puntos

Conectan tres o más equipos de forma directa.

-Líneas Digitales

La transmisión de datos se realiza mediante señales digitales. Esta transmisión utiliza variaciones
de voltaje y siguen códigos determinados.
 
-----------------------------------------------------------------------------
.==========================================================================.
|===========~ 8- El modelo OSI. ~|==========================================
|===========================================================================

Primero hay que definir lo que es un protocolo, un protocolo son un conjunto de reglas que se
establecen para posibilitar la transferencia de datos entre las estaciones. Un modelo estandar
es el modelo OSI. Este modelo tiene una arquitectura de niveles,esto consiste en agrupar funciones
determinadas en partes determinadas o niveles.
En este modelo, el objetivo de cada nivel es poder ofrecer servicios al nivel superior. 
Toda información que se pase de un equipo a otro debe de ir pasando del nivel superior al nivel
inferior pasando por todos los demás niveles de forma descendente.
Consta de 7 niveles:

	1. Nivel físico
	   
	   En este nivel se define la técnica de transmisión, el tipo de conexión. También define
	   la señal utilizada para el paso de datos generados por los niveles anteriores.

	2. Nivel de enlace de datos

	   Este nivel tiene como objetivo garantizar el paso de datos de un equipo a otro a través
           del nivel físico.

	3. Nivel de red

	   En este nivel se direccionan los mensajes y se establecen las direcciones físicas. También
	   elige el camino más adecuado para el traspaso de los paquetes.

	4. Nivel de transporte

	   Detecta los errores ocurridos, hace un informe del estado de la red.

	5. Nivel de sesión

	   Permite la utilización de alguna aplicación que haya en los equipos durante un periodo
	   llamado sesión. 

	6. Nivel de presentación

	   Define el formato utilizado en los mensajes, también se encarga de la seguridad de la red
	   utilizando métodos de encriptación.

	7. Nivel de aplicación

	   Es como una ventana para los procesos que requieren acceder a las posibilidades de la red.
