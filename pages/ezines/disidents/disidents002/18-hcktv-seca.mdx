--------------------------------[1.1a Claves]----------------------------------------------------
-------------------------------------------------------------------------------------------------




El algoritmo SECA se basa en una controlword (cw) de 8 bytes encriptada la cual mediante una 
clave de 16 bytes nos da una controlword de 8 bytes desencriptada.Para esto se utilizan claves 
operaciona les.Este algoritmo es conocido, tambien se puede calcular la firma y llevar a cabo 
la Superencryptacion, pero esta informacion es complicada y para ello hay que saber de 
procedimientos matematicocriptograficos. 

La clave de 16 bytes se compone de una clave primaria de 8 bytes (primary key) y de una 
clave secundaria de 8 bytes (secondary key). La clave secundaria puede ser igual a la primaria.
PK Y SK se identifican mediante los indices de 0x00 al 0x0F, asi tenemos una posibilidad potencial 
de 16 claves.Las claves de indices 0x0C a la 0x0E son las claves operacionales.La clave de indice 
de 0x0F parece ser que tiene una funcionalidad interna especial. Las claves de indices 0x00 a la 
0x0B son las claves de manejo. 

Ademas distinguiremos entre las claves de SECA y las claves del proveedor. Mediante la division 
entre las SECA keys y las Proveedor keys podemos conseguir una estructura de DERECHOS:cada 
proveedor solo puede cambiar sus propias fechas, para cosas a mas alto nivel (como el a±adir o 
borrar proveedores) son necesarias las SECAKEYS.

-----------------------[1.1b CONSTRUCCION DE UNA INSTRUCCION SECA]-------------------------------
-------------------------------------------------------------------------------------------------


El sistenma SECA tiene tres niveles de ordenes.Es importante, el saber distinguir entre ellos:

1----Instruccion:Es una funcion que se construye con un valor determindado en el INS byte
2----Comando: Es una funcion que se construye con un valor determinado, que es el paquete de 
datos enviado a la tarjeta mediante ciertos valores
3----Nanocomando o Nano: Es una funcion que se construye con un valor determinado, que es el 
paquete de datos enviado a la tarjeta mediante ciertos valores.

Se utilizan cabezeras de comandos de 5 bytes segun la norma ISO 7816.Toldos los numeros que se 
utilizan estan en hexadecimal .Una instruccion SECA tiene la siguiente estructura:


                        CLA  INS  P1  P2  (I)LEN/P3  DATA  82  SIG

               CLA=byte de clase
               La unica vclase valida para seca es la C1

               INS=byte de instruccion

               P1=parametros o Byte de referencia
               P2=parametros o Byte de referencia

               ILEN/ LEN / P3=byte de longitud=Longitud del String de Respuesta( en hesadecimalí)

               DATA=parte de datos de la instruccion, no todas las instrucciones utilizan esto
               (contiene Comandos y Nanocomandos, asi como su parte de datos)

               82=Nano 0x82(sigue la firma)

               SIG= firma(se compone de ocho bytes, no es necesario para todas las instrucciones)


En este capitulo solo hablaremos sobre lo que arriba esta dicho( claves e instrucciones en seca)
