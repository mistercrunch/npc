t.02 Como conecto mi Linux a un provider?
-----------------------------------------

Hay muchas formas, el Net2 HOWTO lo explica muuuy bien, y tiene scripts de
ejemplo que pueden usar tranquilamente, este es un ejemplo usando "dip" con una
coneccion telefonica y con el protocolo SLIP (PPP es mas dificil de sacar
andando de una).  La asignacion de ip es dinamica, asi que cada vez que nos
conectemos vamos a tener una direccion distinta...

Primero hay que tener los siguientes datos:

- Nombre de tu maquina (connery.cia.org en el ejemplo)
- Un nombre de usuario
- Un password
- DNS e IP del provider (200.26.39.2 y ba.net en el ejemplo)

Generalmente no hace falta configurar los distintos dispositivos y archivos
para el ruteo de los paquetes (inetd, rc.inet1 y rc.inet2, etc) ya que la
instalacion de linux se encarga de todo eso.

En /etc/inetd.conf se especifican los daemons arrancan con rc.inet2, como
seguridad, aconsejo comentar (ponerle un "#" adelante) las lineas que hablan
 de finger, netstat y systat

Tambien entran en juego los siguientes archivos pero que no necesitan ser
modificados:
/etc/services
/etc/protocols
/etc/host.conf

En /etc/hosts se especifican los nombres y los IPs de las maquinas a las que
nos conectamos (no se ponen TODAS las maquinas, solo el ip y dns de nuestro
provider, despues el se encarga del ruteo), el ip que nos asigna el provider
(como la asignacion es dinamica va 0.0.0.0) y el loopback (127.0.0.1)
Por ejemplo, si nuestro provider fuera el ya famoso Argentina On line
/etc/hosts quedaria:

127.0.0.1       localhost
127.0.0.1       connery.cia.org connery
0.0.0.0         connery.cia.org connery
200.26.39.2     ba.net

connery.cia.org es el nombre de la maquina (/etc/HOSTNAME), connery es un
alias, y ba.net es el dns del provider cuyo ip es 200.26.39.2

En /etc/resolv.conf se indica el nombre del server que se va a encargar de
rutear los paquetes que vayan dirigidos a otras maquinas que no sean las que
se especifican en /etc/host.  Esta formado por dos partes: el ip del
provider y el nombre de nuestra maquina (/etc/HOSTNAME):

domain connery.cia.org
nameserver 200.26.39.2

Con esto termina la configuracion, usen el script del howto y hagan otro para
hacerlo discar e ingresar; ejemplo de script para llamar/entrar:

dip -v -m 496 -p SLIP [nombre_del_script_sacado_del_howto]

y antes que se desesperen....PARA CORTAR pueden:
                                        * Usar kill para matar a dip
                                        * dip -k (que es la misma mierda)
