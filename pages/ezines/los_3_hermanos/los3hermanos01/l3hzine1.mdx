A. Escuchas telefonicas con el Motorola:
-----------------------------------------

Este truco que funciona con telofonos celulares Motorola desde el clasico
PT-500 hasta el moderno StarTAC. No funciona en los ladrillo ni en ninguno
de los modelos digitales.

1. Prende el telefono presionando la clasica (PWR)

2. Marca (FCN) 0 0 * * 8 3 7 8 6 6 3 3 (STO) #

3. Marca 1 1 7 9 9 # 0 8 #

Desde este momento empezas a escuchar lo que la gente habla por telofono.
Podes hablar tranquilamente ya que desde que marcas el paso 2 el telefono
pasa a ser una radio, las teclas SEND y END ya no cumplen funcion alguna.
Lo que escuchas corresponde al canal de radio N° 799. Los canales de radio
corresponden a frecuencias que usan los celulares para establecer la
comunicacion entre el usuario y la base. Asi como el telefono de tu casa
usa un cable hasta la central, tu celular usa un enlace de radio para
establecer la misman conexion.

El paso 1 hace que el telefono funcione, o sea que deje de estar apagado.

El paso 2 habilita en el telefono el modo de servicio. Si te fijas en las
letras del teclado 83786633 representan TESTMODE.

El paso 3 hace dos funciones.
    11799# establece el canal de radio a fijar en la radio.
    08# le dice al receptor que cargue el canal y que abra el amplificador de
     audio.

Hay muchos mas canales en el espectro de radio que funcionan (como el 300),
Para seleccionar otro canal tenes  que repetir el paso 3 con el canal deseado
en reemplazo del 700.

Ejemplos:

El canal 300 seria 1 1 3 0 0 # 0 8 #

El canal 333 seria 1 1 3 3 3 # 0 8 #

Si bien el ruido se escucha, hasta que no cargas el canal en la radio (08#)
no pasa al canal establecido con 11xxx#.

Como habras notado la tecla numeral (#) corresponde al Enter.
Aunque te tientes no marques mas codigos que los aqui explicados. Hay
parametros que no tengo idea que estan y pueden modificar o bloquear la
funcionalidad de tu celular.

                          por |_--=KaNeDa=--_|

4.1 Que son los Nuemros IP?
----------------------------

        Toda computadora que esta conectada a una red, posee una "direccion
virtual", esta direccio esta conformada por X numeros, o sea que cada maquina
que esta conectada a la red posee un numero IP distinto.

        Estos numeros ocupan solamente 4 Bytes y su esquema es:
xxx.xxx.xxx.xxx (cada xxx, pertenece a un numero entre 0 y 255).

        Cada porveedor de internet posee una cantidad x de Numeors IP, que
son utlilizados cada vez que un usuario de ese proveedor se conecta a la red,
cada vez que un usuario se conecta a su proveedor le da un numero IP distinto.


4.2 Que son los DNS?
--------------------

        Todos los sitios Webs estan guardados en un servidor, ests servidores
tambien tienen su propio numero IP.
        En internet existe el servicio DNS ("Dominion name servis", "Servicio de
nombres de dominio"), el DNS consiste en conmbertir o ponerle nombres a las
direcciones IP de los servidores Por ej: tenesmos el IP 216.32.14.52, dicho
IP pertenese al DNS www.yahoo.com.ar

Pero como saber el IP de un servidor? eso es fasil tan solo usando el comando
PING que estan en todas las PC, en el proximo capitulo!

5.1. Comando PING:
------------------

C:\WINDOWS>ping

Uso: ping [-t] [-a] [-n cantidad] [-l tamaño] [-f] [-i TTL] [-v TOS]
          [-r cantidad] [-s cantidad] [[-j lista de host] | [-k lista de host]]
          [-w Tiempo de espera agotado] lista de destino

Opciones:
   -t                 Solicita eco al host hasta ser interrumpido.
                      Para ver estadísticas y continuar: presione Ctrl-Inter.
                      Para interrumpir: presione Ctrl-C.
   -a                 Resuelve direcciones a nombres de host.
   -n cantidad        Cantidad de solicitudes de eco a enviar.
   -l tamaño          Tamaño del búfer de envíos.
   -f                 No fragmentar el paquete.
   -i TTL             Tiempo de vida.
   -v TOS             Tipo de servicio.
   -r cantidad        Registrar la ruta para esta cantidad de saltos.
   -s cantidad        Registrar horarios para esta cantidad de saltos.
   -j lista de hosts  Ruta origen variable en la lista de host.
   -k lista de hosts  Ruta origen estricta en la lista de host.
   -w tiempo          Tiempo de espera agotado de respuesta en milisegundos.

Que es Telnet? 
--------------

    Bueno Telnet es una de las aplicaciones mas potentes, aunque trabaja en
modo texto, se emplea principalmente para acceder a grandes bases de datos,
con poderosos ordenadores que trabajan incansablemente para nosotros.

    Se basa en convertir nuestro ordenador en un terminal de un potente 
ordenador que administra una base extensisima de informacion, de tal forma 
que desde nuestra casa o lugar de trabajo, con un simple ordenador 
domestico (como por ejemplo un 8086 con solo 256 kb de RAM) podemos acceder a 
la base de datos de la biblioteca de la Universidad de Madrid (o de 
Nueva York, pues la distancia no es problema) controlada por una estacion 
de 10 unidades centrales de proceso y 512 Megabytes de RAM. Lo 
importante de convertirnos en un terminal es que todo el trabajo lo realiza la 
maquina a la que nos conectamos, y no importa el sistema operativo que 
estΘ usando, pues la aplicacion TELNET que estemos utilizando emulara 
satisfactoriamente el proceso de comunicacion. Es por esto que TELNET se 
utiliza principalmente para la consulta de bases de datos de cualquier 
tipo. 

Para conectarnos debemos introducir el comando connect nombre_maquina 
puerto, tambien es posible que te bajes un cliente telnet como 
SecureCRT, en este solo tendras que poner el ip o hostname del la maquina
remota y un nombre de usuario! (desde ms-dos tambien puedes abrir telnet, solo 
tienes que tipiar: "telnet nombredelapagina.com" y esto te abrira 
telnet. La mßquina a la que hacemos telnet nos pedira un login (palabra para 
iniciar la conexion) que no es una contraseña, y que depende de cada 
maquina. Asi pues, cuando usted conozca la direccion de una maquina que 
ofrece un interesante servicio de informacian telnet, intente tambien 
averiguar cual es su login para poder acceder. Hay otros servidores de 
telnet que solo dan informacion a los usuarios que tienen una cuenta
